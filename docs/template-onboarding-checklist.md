# OctoAcme Team Member Onboarding Checklist

This template provides a standardized onboarding checklist for new team members joining an OctoAcme project. Customize sections based on the specific role and project needs.

---

## Pre-Start Preparation

### Administrative Setup
- [ ] Employee/contractor paperwork completed
- [ ] IT equipment ordered and shipped
- [ ] Account access requests submitted
  - [ ] Email and calendar
  - [ ] GitHub organization and repositories
  - [ ] Project management tools (Jira, Azure DevOps, etc.)
  - [ ] Communication platforms (Slack, Teams, etc.)
  - [ ] Development environment tools

### Documentation & Resources
- [ ] Welcome email sent with first-day logistics
- [ ] Team roster and org chart shared
- [ ] Project documentation repository access provided
- [ ] Onboarding buddy/mentor assigned

---

## Week 1: Orientation & Access

### Day 1
- [ ] Welcome meeting with manager
- [ ] Team introductions and meet & greet
- [ ] Workspace/desk setup completed
- [ ] Verify all account access is working
- [ ] Review company policies and code of conduct
- [ ] Schedule regular 1:1s with manager

### Days 2-5
- [ ] Review project management documentation in `docs/` folder
  - [ ] [Project Management Overview](octoacme-project-management-overview.md)
  - [ ] [Roles & Personas](octoacme-roles-and-personas.md)
  - [ ] [Project Initiation Guide](octoacme-project-initiation.md)
  - [ ] [Project Planning](octoacme-project-planning.md)
- [ ] Clone relevant repositories and set up development environment
- [ ] Run local build and test suite successfully
- [ ] Attend daily standups and team meetings
- [ ] Shadow team members in their daily work
- [ ] Review recent sprint retrospective notes

---

## Week 2: Deeper Dive

### Technical Onboarding
- [ ] Review system architecture documentation
- [ ] Understand CI/CD pipeline and deployment process
- [ ] Review coding standards and best practices
- [ ] Set up local development tools (linters, formatters, debuggers)
- [ ] Complete first "good first issue" or pair programming session

### Process Familiarization
- [ ] Attend sprint planning meeting
- [ ] Review backlog and understand prioritization process
- [ ] Understand definition of done and acceptance criteria
- [ ] Learn bug reporting and tracking process
- [ ] Review [Execution & Tracking](octoacme-execution-and-tracking.md) processes

### Team Integration
- [ ] Lunch or coffee chat with team members
- [ ] Meet with onboarding buddy for Q&A
- [ ] Introduction meeting with key stakeholders
- [ ] Join relevant Slack/Teams channels

---

## Weeks 3-4: Contribution & Independence

### Active Contribution
- [ ] Complete first pull request and code review
- [ ] Participate in code reviews for others
- [ ] Take ownership of assigned user stories/tasks
- [ ] Contribute to team discussions and decisions
- [ ] Attend sprint retrospective and provide feedback

### Expanded Knowledge
- [ ] Review [Release & Deployment Guide](octoacme-release-and-deployment.md)
- [ ] Understand risk management and escalation process
- [ ] Learn about quality assurance and testing standards
- [ ] Review documentation on stakeholder communication
- [ ] Understand continuous improvement practices

### Feedback & Adjustment
- [ ] 30-day check-in with manager
- [ ] Provide feedback on onboarding experience
- [ ] Identify any gaps in knowledge or access
- [ ] Set initial performance goals and expectations

---

## Role-Specific Onboarding

### For Developers
- [ ] Review technical debt register
- [ ] Understand branching and PR workflow
- [ ] Learn deployment procedures and rollback process
- [ ] Set up monitoring and observability tools
- [ ] Review security and compliance requirements

### For QA/Testing Engineers
- [ ] Review test strategy and test plans
- [ ] Set up test automation framework
- [ ] Understand regression testing process
- [ ] Learn defect tracking and reporting workflow
- [ ] Review quality metrics and dashboards

### For Product/Project Managers
- [ ] Review product roadmap and backlog
- [ ] Meet with stakeholders and understand their needs
- [ ] Understand budget and resource constraints
- [ ] Review reporting and status update processes
- [ ] Learn project tracking tools and dashboards

### For Tech Leads
- [ ] Review architectural decision records
- [ ] Understand technical strategy and roadmap
- [ ] Meet with engineering leadership
- [ ] Review code review and mentoring practices
- [ ] Understand technical debt prioritization

---

## 90-Day Milestones

- [ ] Fully independent in primary role responsibilities
- [ ] Completed significant feature or project contribution
- [ ] Established working relationships with team and stakeholders
- [ ] Providing value in team ceremonies and discussions
- [ ] 90-day performance review completed
