# Role Onboarding Checklist

This checklist helps new team members understand their role and get up to speed quickly on OctoAcme project management processes.

---

## General Onboarding (All Roles)

- [ ] Review the [Project Management Overview](./octoacme-project-management-overview.md)
- [ ] Read the [Roles & Personas](./octoacme-roles-and-personas.md) document
- [ ] Review the [Responsibility Matrix](./responsibility-matrix.md) to understand role interactions
- [ ] Identify your primary stakeholders and schedule introductory meetings
- [ ] Join relevant communication channels (Slack, Teams, email lists)
- [ ] Get access to project boards, dashboards, and repositories
- [ ] Attend your first standup, sprint planning, or team sync meeting
- [ ] Review the current project charter and active initiatives

---

## Developer Onboarding

- [ ] Set up development environment and verify access to repositories
- [ ] Review coding standards and contribution guidelines
- [ ] Understand the Definition of Done for code changes
- [ ] Learn the PR review process and required approvals
- [ ] Review the CI/CD pipeline and automated testing setup
- [ ] Identify your PM, Product Manager, and QA Specialist contacts
- [ ] Schedule pair programming session with a senior developer
- [ ] Read [Execution & Tracking](./octoacme-execution-and-tracking.md) for workflow details

---

## Product Manager Onboarding

- [ ] Review the product roadmap and current priorities
- [ ] Understand the backlog structure and prioritization criteria
- [ ] Meet with stakeholders to understand business objectives
- [ ] Review success metrics and KPIs with the Data Analyst
- [ ] Learn about user research findings and customer feedback channels
- [ ] Review [Project Planning](./octoacme-project-planning.md) processes
- [ ] Understand release cadence and planning cycles
- [ ] Connect with UX Lead to review design work in progress

---

## Project Manager Onboarding

- [ ] Review active projects and current status
- [ ] Understand the risk management process and current risk register
- [ ] Learn the escalation paths and stakeholder communication plan
- [ ] Review [Project Initiation](./octoacme-project-initiation.md) guidelines
- [ ] Understand reporting requirements and cadence
- [ ] Meet with Product Managers to align on priorities
- [ ] Review dependency tracking and cross-team coordination methods
- [ ] Access project tracking tools and dashboards

---

## UX Lead Onboarding

- [ ] Review the design system and component library
- [ ] Understand the user research process and current findings
- [ ] Access design tools (Figma, Sketch, etc.) and repositories
- [ ] Review accessibility standards and compliance requirements
- [ ] Schedule design reviews with Product Managers and Developers
- [ ] Understand the usability testing process
- [ ] Review current prototypes and design work in progress
- [ ] Connect with QA Specialist for usability validation processes

---

## QA Specialist Onboarding

- [ ] Review test plans and current test coverage
- [ ] Access test management tools and defect tracking systems
- [ ] Understand the Definition of Done and acceptance criteria format
- [ ] Learn the bug triage process and severity definitions
- [ ] Review [Execution & Tracking](./octoacme-execution-and-tracking.md) for QA workflows
- [ ] Set up test environments and automation frameworks
- [ ] Schedule meetings with Developers to understand architecture
- [ ] Connect with UX Lead for usability testing coordination

---

## Data Analyst Onboarding

- [ ] Access analytics platforms and data warehouses
- [ ] Review current dashboards and reporting structure
- [ ] Understand success metrics defined for active projects
- [ ] Learn data governance and privacy requirements
- [ ] Schedule meetings with Product Managers to understand priorities
- [ ] Review instrumentation standards and data collection practices
- [ ] Understand A/B testing frameworks and experimentation processes
- [ ] Connect with Developers to discuss data pipeline architecture

---

## Change Manager Onboarding

- [ ] Review current release plans and timelines
- [ ] Understand stakeholder groups and communication channels
- [ ] Access training platforms and documentation systems
- [ ] Review [Release & Deployment Guide](./octoacme-release-and-deployment.md)
- [ ] Learn the change impact assessment process
- [ ] Schedule meetings with Project Managers for release coordination
- [ ] Review adoption metrics and feedback collection methods
- [ ] Understand training material creation and delivery processes

---

## Follow-up (All Roles)

- [ ] Schedule 30-day check-in with your manager
- [ ] Provide feedback on the onboarding process
- [ ] Identify any documentation gaps or unclear processes
- [ ] Confirm you have all necessary access and tools
- [ ] Share what you learned with the team to improve onboarding
